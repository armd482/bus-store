# 정답지 백테스트 — 무엇을 어떻게 실측하나

`score.py`(§9 5·6 최소 구현)를 돌리려면 아래 3종의 입력이 필요하다.
엔진·GTFS·그래프빌드 없이 **횡단보도 데이터 + 실측 라이드**만으로 지표2(정확도)를 친다.

| 파일 | 내용 | 얻는 법 |
|---|---|---|
| `ground_truth.json` | S1~S5 실측 라이드 (출발시각·도보경로·실제 탄 연결) | **직접 라이드** (§1) |
| `crosswalks.json` | 경로에 걸린 횡단보도의 거리·주기·녹색 | **현장 스톱워치 + OSM/표준데이터** (§2) |
| (timetable, ground_truth 안) | 탑승 정류장의 배차 시각 | GTFS/PDF/수집기 이력 (§3) |

핵심: **"어느 연결(버스/열차)을 잡았나"** 가 Sm 이 맞히려는 대상이다. 도착 시각이 아니라
**탄 차의 출발시각**을 실측해야 한다 (문서 line 1218 — 도착 분산에 뭉개지 말 것).

---

## 1. 정답지 라이드 실측 (S1~S5) — 반나절

각 시나리오 = "한 번의 실제 통근/이동". 통근 경로 하나를 3~4개 시간대·요일에
반복하면 S1~S5 가 나온다. 라이드 한 번에 스마트폰으로 아래를 기록한다.

### 재는 값 (한 라이드당)
1. **출발시각** `depart` — 출발지(집·역 출구)를 **발을 떼는 순간**의 시계 시각. `08:05`.
2. **탄 연결** `observed_connection` — 실제 탑승한 차의 **출발시각**.
   - 버스: 정류장 안내기/앱의 그 차 도착·출발 시각, 또는 승차 순간 시각.
   - 열차: 승강장 전광판 출발 시각.
   - 이것이 Sm 이 맞혀야 할 정답. `08:25`.
3. **도보 경로** `walk.segments` — 출발지→탑승정류장 경로를 **횡단보도 기준으로 분절**.
   - 각 구간 `dist_m`: 지도(카카오맵/OSM)에서 잰 구간 거리.
   - 각 구간이 지난 `crossings`: `crosswalks.json` 의 id 목록.
   - 예: `집 →220m→ [횡단 cw_A] →200m→ [횡단 cw_B, cw_C] →정류장`.
4. **고정시간** `fixed_s` (선택) — 개찰·계단·환승통로 등 신호와 무관한 고정 지연(초).
   신호 검증이 목적이므로 이건 **분리**해서 상수로 넣는다 (계획서 교정 ③: 신호와 계단 분리).
5. **본인 도보 속도** — GPS 앱(예: 만보기/러닝앱)이 준 그 라이드의 평균 속도(m/s).
   `score.py` 상단 `USER_SPEED` 를 이 값으로 교체. (없으면 거리÷도보소요로 역산.)

### 팁
- **같은 OD 를 여러 요일·시간대**에 — 신호 위상·배차가 달라 분화(플립) 사례가 잡힌다.
- 라이드 중 **각 횡단보도에서 신호를 기다렸는지/바로 건넜는지**를 메모하면, 나중에
  Sm 의 기대대기가 그 라이드에서 맞았는지 사후 대조할 수 있다 (선택).
- S4 처럼 "빠듯하게 차를 놓쳤/잡았다" 싶은 라이드가 **금맥** — 거기서 S15/Sm 이 갈린다.

## 2. 횡단보도 실측 (crosswalks.json) — 반나절, 계획서가 인정한 유일한 신규 데이터

경로에 걸린 횡단보도 **3~4개만**. 각 횡단보도에 대해:

### 2-1. 횡단거리 `dist_m` — 세 방법 중 택1 (정확도 순)
1. **현장 실측**: 보도 연석~연석을 걸음/줄자로. 가장 정확.
2. **OSM**: `pangyo.osm.pbf` 의 도로 폭 태그(`width`) 또는 차로수×3.5m. 무료·즉시.
3. **Tmap 보행자 API** `facilityType=15`: 횡단보도 위치가 나온다.
   `spike/scripts/tmap_pedestrian_parallel.py` 에 호출·키로딩 코드가 이미 있으니 재사용.

### 2-2. 주기 `cycle` · 보행녹색 `green` — 현장 스톱워치 프로토콜
**표준데이터포털**(경찰청 보행신호 데이터)에 있으면 그걸 쓴다. 없으면 현장에서 잰다:

```
스톱워치 프로토콜 (횡단보도당 ~5분):
  1. 보행신호가 "녹색으로 바뀌는 순간" 랩 시작.
  2. "녹색 점멸 시작"까지 = green (보행 녹색 시간). 기록.
     (엄밀히는 정적 녹색만. 점멸을 포함할지는 §7.4 각주 — 보수적으로 정적 녹색만.)
  3. 다음 "녹색으로 바뀌는 순간"까지 = cycle (전체 주기). 기록.
  4. 3주기 반복해 평균. 편차가 크면(감응식 신호) 메모 — 그 횡단보도는 모델 신뢰도 낮음.
```

- **red = cycle − green** 은 코드가 자동 계산. red 만 따로 안 재도 된다.
- 감응식(차량 감지)·보행자 버튼 신호는 주기가 변동 → `_note` 에 표시하고 결과 해석 시 주의.
- **설계속도 1.0 m/s 주의**(문서 §7.4): green 은 신호기가 정한 값이지 내가 빨리 걷는다고
  줄지 않는다. `USER_SPEED` 는 **횡단 소요와 통과판정**에만 쓰이고 green 산정엔 안 쓰인다.

### 2-3. 채우기
```json
"cw_A": {"dist_m": 18, "cycle": 95, "green": 20}
```

## 3. 시각표 (timetable) — 수집기 이력이 이미 있다

탑승 정류장에서 그 노선의 **출발 시각 배열**. `ground_truth.json` 의 각 시나리오 안에.
- 버스: `bus_collector` 가 쌓은 도착정보 이력, 또는 정류장 시각표.
- 열차: 15098251 시각표 / 신분당선 PDF (문서 §3.3.1).
- **시각 근접 매칭**으로 쓴다 (문서 line 1236 — 열차번호 조인은 신분당선에서 불가).

```json
"timetable": ["08:14", "08:25", "08:36"]
```

## 4. 돌리고 읽기

```bash
cd spike/validation
python3 score.py                 # 기대값 모델 — 지표2 기본
python3 score.py --worst         # 최악값(적색 전체)도 — 둘 다 갈려야 진짜(§7.4)
python3 score.py --sweep         # 속도별 catch/miss 뒤집힘 위상 % (§7.4 물리 직접)
python3 score.py --buffer 20     # 탑승 안전버퍼 20초로 민감도 확인
```

### 결과 판정
- **지표2 통과** = 분화한(예측 갈린) 사례에서 **Sm 승 > S15 승**.
- **기대값에서도 이겨야** 진짜(문서 line 1168). `--worst` 에서만 갈리면 인공물 → 기각.
- 분화가 0 건이면 표본 부족 — S1~S5 를 더 채우거나(요일·시간대), 빠듯한 라이드를 노려라.

## 5. 반드시 피할 함정 (문서 자기교정)

1. **베이스라인은 S15(15초)** 지 S0(0초) 아님 (line 1138). S0 대비는 부풀림.
2. **기대값(20s급)으로 판정**, 최악값(60s급)은 검증용 병행 (line 1164~1168).
3. **신호와 계단 분리** — 계단·개찰은 `fixed_s` 로 빼고 신호만 본다.
4. **크기 말고 빈도** — Step 1(OTP 배치)에서 분화가 `버스→도보→지하철` 조합에
   몰리는지 확인 (지표3, 문서 §9).
5. **도착 시각이 아니라 탄 차의 출발시각**을 실측 (line 1218).

## 6. 모델 노트 — 왜 기대대기에 횡단소요를 더했나

문서 §7.1 은 기대대기 = `red²/(2·cycle)`. `score.py` 는 이를 속도-인지로 일반화한다:

```
E[wait] = (red + 횡단거리/속도)² / (2·cycle)
```

유도: 유효녹색 창 = `green − 횡단거리/속도`(§7.4). 이 창 밖에 도착하면 다음 녹색까지
대기하는데, 그 "대기해야 하는 위상 구간"이 `red + 횡단소요` 로 넓어진다. 균일 위상에
적분하면 위 식. 횡단소요→0 이면 §7.1 의 `red²/(2·cycle)` 로 정확히 환원된다.
즉 **§7.1(기대대기)과 §7.4(속도가 창을 가른다)의 통합** — 속도가 도보시간뿐 아니라
대기에도 들어가 차별점 메커니즘이 기대값에 반영된다.
```
