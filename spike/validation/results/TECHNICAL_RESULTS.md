# 기존 기술 Validator 결과

평가 프로토콜:
[`MEASUREMENT.md`](../MEASUREMENT.md)

이 문서는 동일한 Case 자료를 기존 기술 평가기에도 입력한 결과를 기록한다.
경량 검증과 판정 기준·입력 출처를 섞지 않는다.

## 비교 모델

- `Naver`
- `Speed`
- `Speed+Signal`
- `Oracle`
- `TMAP-Personal`
- `TMAP+Signal`
- `TMAP-Oracle`
- `Kakao-Personal`
- `Kakao+Signal`
- `Kakao-Oracle`

API·좌표·주기·실시간 신호가 없어서 실행하지 못한 모델은 실패값이나 0초가
아니라 `unavailable`로 기록한다.

## 실행 원칙

- 채팅 관측을 먼저 `ground_truth.json` 형식으로 정규화한다.
- `source: real`, `split: test`로 기록한다.
- 캡처에서 확인할 수 없는 값을 추정해서 정답으로 넣지 않는다.
- 실제 신호 대기는 `Oracle`에만 사용한다.
- TMAP·카카오·실시간 신호의 성공 여부와 커버리지를 Case마다 기록한다.
- API 결과를 보고 개인 시간비율이나 안전마진을 변경하지 않는다.
- 기술 모델의 종합 판정은 유효한 `test` 연결 이벤트 3건 이상에서만 한다.
